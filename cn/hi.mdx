# 使用统计

移动代理流量使用统计

移动代理流量使用详情可以在您的仪表板统计页面上找到。只需通过仪表板左侧面板导航到移动代理页面，然后点击**[使用统计](https://dashboard.decodo.com/mobile-proxies/statistics?)**部分。

<Callout icon="📘" theme="info">
  如果您想了解流量是如何计算的，请前往**[流量使用](https://help.decodo.com/docs/traffic-usage)**。
</Callout>

***

## 统计数据折线图

您将看到一个折线图，显示您使用代理发送的总**流量**或**请求**。

<Callout icon="📘" theme="info">
  使用数据每 10 到 15 分钟更新一次。统计数据以 UTC 时区提供。
</Callout>

* 在**流量**折线图中，纵轴显示以 `MB` 和/或 `GB` 为单位的已用流量，而横轴显示您分组的时间。
* 您可以将鼠标悬停在折线图中的数据点上，以查看所选时间的详细使用情况。

* 在**请求**折线图中，纵轴显示使用代理发送的请求数量，而横轴显示您分组的时间。

## 统计数据表

1. 您可以找到**日期**、**流量使用**、**下载**和**上传**流量（以 `MB` 或 `GB` 为单位）。接下来的两列描述了**请求**数量和**支出**。**支出**是相当于所用流量的美元成本。

## API

您还可以通过 **[API](https://help.decodo.com/reference/public-api-key-authentication)** 免费检索代理流量和使用详情。

### 获取流量

以下 `POST` 端点将允许您接收已用流量信息：

`https://api.decodo.com/api/v2/statistics/traffic`

您可以根据您希望接收的流量调整请求参数。要查找所有请求参数的列表，请前往 **[获取流量](https://help.decodo.com/reference/get-traffic)** API 文档页面。

在链接的 API 文档页面中，还有各种编程语言的请求示例。此外，您可以在调整 **Body Params** 部分后，在右侧的 **Authorization** 字段中输入您的 API 密钥来测试请求。

以下是通过 **获取流量** 端点的 `cURL` 请求示例。如果您想尝试该请求，请确保在第 3 行中替换 `<your_authorization_API_key>`。

```curl
curl --request POST \
     --url https://api.decodo.com/api/v2/statistics/traffic \
     --header 'Authorization: <your_authorization_API_key>' \
     --header 'accept: application/json' \
     --header 'content-type: application/json' \
     --data '
{
  "proxyType": "mobile_proxies",
  "startDate": "2026-01-01 00:00:00",
  "endDate": "2026-01-03 00:00:00",
  "groupBy": "day",
  "limit": 500,
  "page": 1,
  "sortBy": "grouping_key",
  "sortOrder": "asc"
}
'
```

## 统计中无法识别的使用情况

### 不熟悉的目标网站

如果您在统计页面上看到不熟悉的目标，请记住它可能是您定位的网站的**子域**。这意味着目标网站使用子域来组织内容或服务的不同部分。仪表板也会**跟踪**这些**子域**。

### 意外的流量消耗

* 如果您注意到代理的流量使用量较高，请注意，流量可能是在您使用代理时在**后台**运行和更新的**应用程序**消耗的。
* 此外，**目标网站**可能已**更改**并引入了新**内容**，这会影响代理流量。
* 通常，建议定期为您的**代理用户**生成新**密码**。
* 不要忘记在使用完代理后**关闭**您用于运行代理的**工具**或**程序**。

查看 **[高流量使用](https://help.decodo.com/docs/high-traffic-usage)** 以更详细地了解流量差异。
